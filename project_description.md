# Project Description: Automation Workspace

## نظرة عامة (Overview)
هذا المشروع عبارة عن تطبيق ويب تفاعلي (Web-based Automation Workspace) يسمح للمستخدم بإنشاء مسارات عمل (Workflows) عن طريق السحب والإفلات (Drag and Drop). 
الهدف الأساسي هو محاكاة بيئة أتمتة بسيطة ومرئية.

## الإجابة على التساؤلات:
1. **هل نحتاج إلى Backend (خادم)؟**
   - **الجواب: لا، ليس ضرورياً لهذه الوظيفة الفردية.** يمكننا محاكاة وتنفيذ كل العمليات (Execution) بالكامل في المتصفح (Frontend) باستخدام React. هذا سيجعل المشروع أسرع في البناء وأسهل في العرض والتقييم دون الحاجة لإعداد خوادم أو قواعد بيانات. سنعتمد على الحفظ المحلي (Local Storage) لضمان عدم ضياع العمل عند تحديث الصفحة.
   
2. **ما هو الخرج (Output) المتوقع لتطبيق الأتمتة هذا؟**
   - **الجواب:** الخرج سيكون عبارة عن "شاشة عرض" أو "Console" داخل التطبيق. عند الضغط على زر "تشغيل - Run"، سيقوم التطبيق بقراءة مسار العمل (Workflow) الذي قمت بوصل عُقده ببعضها، ثم ينفذها.

## المكونات الأساسية (Core Nodes):
- **عقدة النص (Log Node):** 
  عقدة تحتوي على حقل إدخال (Input). يكتب المستخدم بداخلها النص المراد طباعته (مثلاً: "Hello World" أو قراءة مجردة).
- **عقدة اللون (Color Node):**
  عقدة تحتوي على منتقي ألوان (Color Picker) ليختار منه المستخدم لوناً محدداً.

## كيفية عمل التطبيق والتوصيل (Logic):
- يمكن للمستخدم سحب أيقونة Log وأيقونة Color إلى مساحة العمل (Canvas) والربط بينهما.
- **آلية التنفيذ:** إذا قام المستخدم بربط `Color Node` مع `Log Node`، فإن الخرج (Output) الذي سيُطبع في الـ Console الخاص بالتطبيق سيأخذ لون الـ Color Node. إذا لم تكن العقدة متصلة بلون، ستتم الطباعة باللون الافتراضي (الأبيض أو الأسود).

## التقنيات المستخدمة (Tech Stack):
- **Frontend Framework:** React.js (Vite)
- **Drag & Drop Library:** React Flow (وهي مكتبة احترافية جداً لبناء الـ Node-based Interfaces)
- **Styling:** Tailwind CSS (لبناء واجهة عصرية وجذابة تعكس احترافية הـ Vibe Coding)

هذا الوصف يجعل المشروع معقولاً، احترافياً جداً كالتكليف الجامعي، ولا يتطلب تعقيد الـ Backend.
